# Startup Profit Regression with FCNN and LSTM

Лабораторная работа №1 по разведочному и регрессионному анализу данных на основе нейросетевых моделей.

Проект посвящён прогнозированию прибыли стартапов по данным о расходах на исследования и разработки, административные расходы, маркетинг и регион компании. Основная задача — построить регрессионную модель для целевого признака `Profit` и сравнить качество разных подходов к подготовке данных и обучению нейросетевых моделей.

## Рекомендуемое название репозитория

```text
startup-profit-regression-fcnn-lstm
```

## Основной файл проекта

```text
ЛР1_ИАДНОММО.ipynb
```

## Описание задачи

Вариант 3 лабораторной работы связан с набором данных о прибыли стартапов. Необходимо построить модель машинного обучения, которая предсказывает значение `Profit` на основе остальных входных признаков.

В датасете используются следующие признаки:

- `R&D Spend` — расходы на исследования и разработки;
- `Administration` — административные расходы;
- `Marketing Spend` — расходы на маркетинг;
- `State` — штат, в котором работает компания;
- `Profit` — прибыль компании, целевая переменная.

В рамках работы дополнительно рассматриваются преобразования данных, отбор признаков, кодирование категориальных переменных, нормализация и подбор гиперпараметров.

## Цель работы

Цель проекта — провести полный цикл подготовки данных и построения регрессионной модели:

- загрузить и изучить исходный датасет;
- выполнить разведочный анализ данных;
- обработать пропущенные значения;
- провести корреляционный анализ;
- создать дополнительный признак `Total Spend`;
- отобрать существенные признаки;
- преобразовать категориальные признаки в числовой формат;
- выполнить нормализацию данных;
- разделить данные на обучающую, валидационную и тестовую выборки;
- обучить и сравнить FCNN и LSTM;
- выбрать лучшую модель;
- провести GridSearch для подбора гиперпараметров;
- оценить качество модели до и после подбора параметров.

## Используемый датасет

В ноутбуке используется файл:

```text
V3.csv
```

Файл загружается из Google Drive с помощью библиотеки `gdown`.

В исходном наборе данных содержится 50 наблюдений. Датасет включает финансовые показатели стартапов и целевой показатель прибыли.

## Этапы работы

### 1. Загрузка библиотек

В начале ноутбука подключаются библиотеки для работы с данными, визуализации, машинного обучения и построения нейросетевых моделей:

- `pandas`;
- `numpy`;
- `matplotlib`;
- `seaborn`;
- `scikit-learn`;
- `torch`;
- `gdown`.

### 2. Загрузка данных

Данные загружаются из Google Drive в файл `V3.csv`, после чего считываются в `pandas DataFrame`.

После загрузки выполняется первичный просмотр данных с помощью `head()`, `info()` и `describe()`.

### 3. Разведочный анализ данных

На этапе разведочного анализа изучаются:

- структура датасета;
- типы признаков;
- количество пропущенных значений;
- распределения числовых признаков;
- возможные выбросы;
- связи между признаками;
- особенности целевой переменной `Profit`.

В ходе анализа выявлено, что признак `Administration` содержит большое количество пропущенных значений, а признаки `R&D Spend` и `Marketing Spend` имеют высокую вариативность.

### 4. Визуализация данных

Для анализа распределений и зависимостей используются:

- гистограммы;
- графики плотности;
- pairplot;
- тепловая карта корреляций.

Визуализация помогает оценить форму распределений и увидеть возможные линейные зависимости между расходами и прибылью.

### 5. Очистка данных

Пропущенные значения в числовых признаках заполняются медианой с помощью `SimpleImputer`.

Такой подход позволяет сохранить строки датасета и снизить влияние выбросов по сравнению с заполнением средним значением.

### 6. Корреляционный анализ

В работе рассчитывается корреляционная матрица для числовых признаков.

По результатам анализа были выявлены следующие зависимости:

- `R&D Spend` имеет очень сильную положительную связь с `Profit`;
- `Marketing Spend` также заметно связан с `Profit`;
- `Administration` показывает слабую связь с целевой переменной.

Корреляционный анализ используется для понимания влияния признаков и дальнейшего отбора существенных переменных.

### 7. Создание нового признака

В ноутбуке создаётся дополнительный признак:

```text
Total Spend = R&D Spend + Administration + Marketing Spend
```

Этот признак отражает общий объём расходов компании и используется для проверки того, улучшает ли агрегированный показатель качество модели.

### 8. Отбор существенных признаков

Отбор признаков выполняется двумя способами:

- на основе корреляции с целевой переменной;
- с помощью `SelectKBest` и критерия `f_regression`.

В результате анализа наиболее значимым признаком оказался `R&D Spend`. Также существенную роль играет `Marketing Spend`.

### 9. Кодирование категориальных признаков

Категориальный признак `State` преобразуется в числовой формат с помощью One-Hot Encoding.

Для уменьшения мультиколлинеарности используется параметр `drop='first'`, из-за чего одна из категорий штатов удаляется и используется как базовая.

### 10. Преобразование и нормализация данных

В лабораторной работе формируются несколько наборов данных:

- `original` — исходный набор данных;
- `transformed_original` — исходный набор после нормализации;
- `built` — набор с отобранными или построенными признаками;
- `transformed_built` — построенный набор после нормализации.

Для нормализации используется `Normalizer`, который приводит строки признаков к единичной L2-норме.

### 11. Разделение данных

Каждый набор данных делится на три части:

- обучающая выборка — 60%;
- валидационная выборка — 20%;
- тестовая выборка — 20%.

Обучающая выборка используется для обучения моделей, валидационная — для сравнения качества и выбора лучшего варианта, тестовая — для финальной оценки.

### 12. Обучение моделей

В ноутбуке реализованы и сравниваются две нейросетевые модели:

- `FCNN` — полносвязная нейронная сеть;
- `LSTM` — рекуррентная нейронная сеть.

FCNN используется как основная архитектура для регрессии на табличных данных. LSTM добавлена по варианту лабораторной работы и используется для сравнения.

### 13. Сравнение качества моделей

Качество моделей оценивается по метрикам:

- `RMSE` — среднеквадратичная ошибка;
- `R²` — коэффициент детерминации.

По результатам экспериментов лучшей моделью стала FCNN, обученная на исходных данных без нормализации.

LSTM показала низкое качество, так как такая архитектура хуже подходит для статических табличных данных без временной структуры.

### 14. Финальная оценка лучшей модели

Лучшая модель была проверена на тестовой выборке.

Полученные результаты:

```text
Test RMSE: 16407.67
Test R²: 0.865
```

Значение `R² = 0.865` показывает, что модель объясняет около 86.5% дисперсии целевой переменной на тестовых данных.

### 15. GridSearch

Для FCNN был выполнен подбор гиперпараметров с помощью `GridSearchCV`.

Подбирались следующие параметры:

- количество нейронов в скрытых слоях;
- скорость обучения;
- конфигурация архитектуры сети.

После GridSearch качество модели немного ухудшилось:

```text
До GridSearch:
RMSE: 16407.67
R²: 0.86

После GridSearch:
RMSE: 17140.15
R²: 0.85
```

Это показывает, что подбор гиперпараметров не всегда приводит к улучшению, особенно на небольших наборах данных.

## Основные результаты

В ходе работы были получены следующие выводы:

- расходы на исследования и разработки являются наиболее важным фактором для прогнозирования прибыли;
- маркетинговые расходы также имеют заметную связь с прибылью;
- административные расходы имеют слабую индивидуальную корреляцию с `Profit`;
- FCNN лучше подходит для данной задачи, чем LSTM;
- нормализация в этой задаче ухудшила качество модели;
- исходный набор данных без преобразований дал лучший результат;
- GridSearch не улучшил итоговое качество модели;
- для небольших табличных датасетов простая архитектура может оказаться устойчивее сложной настройки.
